Scoring Logic

One prompt. Multiple models. One governed standard.

Norynthe does not ask which answer feels best. It asks which answer is most defensible across a common trust framework, then records why.

Input

Shared benchmark context

The same prompt is routed to multiple models so differences in omission, framing, and structure can be observed directly.

Evaluation

Six scored dimensions + one diagnostic signal

Trust is decomposed into grounding, evidence strength, consistency, context integrity, interpretive stability, and auditability, with cross-model alignment retained as a comparative diagnostic.

Signal

Final Norynthe Score

Dimension scores are weighted, resolved, and translated into a clear trust band without hiding the underlying rationale.

Architecture

Scoring pipeline with built-in challenge.

Norynthe’s MVP does not treat disagreement as noise. The scorer and critic structure makes scoring more defensible by forcing each dimension result through evidence and review.

01
Prompt ingestion Same prompt, selected models, raw outputs preserved.
02
Shared analysis context Normalized payload with prompt, target output, peer outputs, metadata, and rubric.
03
Dimension scorer Initial score, score band, evidence, deductions, and confidence.
04
Dimension critic Rubric review, evidence challenge, score revision or approval.
05
Resolver Deterministic application logic finalizes the score and flags escalations.
06
Aggregation + summary Weighted trust score, band, and an action-ready explanation.

Workbook

The control layer behind the score.

The workbook keeps the system from becoming arbitrary. It defines what each dimension means, which signals count toward the public score, how deductions work, what anchor examples look like, and how the rubric evolves over time.

Rubric

Dimension definitions

Each trust dimension has a definition, role, core question, required checks, evidence rules, scoring bands, common deductions, and anchor examples.

Calibration

Prompt benchmark suite

Versioned revision log

Infrastructure Thesis

          
Why owned compute strengthens the company.

          
If Norynthe becomes a real trust infrastructure layer, owned compute becomes a strategic advantage. High-end local systems matter for control over the evaluator stack, protection of core intellectual property, privacy, long-run economics, and independence from external model providers.

        

        

          

            
Why

            
Control over the evaluator stack

            
DGX-class systems would let Norynthe self-host parts of the scorer and critic pipeline instead of depending entirely on third-party APIs. That matters if evaluator behavior becomes core intellectual property.

          

          

            
Why

            
Lower long-run inference cost

            
If Norynthe is running many dimensions across many models and prompts, evaluation traffic compounds quickly. Owning serious compute can reduce recurring cost once the scoring architecture is stable enough to justify it.

          

          

            
Why

            
Privacy and regulated workflows

            
Some enterprise and institutional use cases will want local processing for sensitive prompts, internal outputs, or regulated reviews. DGX-class infrastructure creates a path toward that without changing the scoring design.

          

        

        

          

            
Why

            
Provider independence

            
The long-term asset in Norynthe is the rubric, calibration process, and resolution logic. Running evaluator models on owned hardware reduces dependency on a single API vendor and gives the system more strategic durability.

          

          

            
Strategic

            
Own the evaluator layer

            
If Norynthe is defining the trust standard, it should have a path to owning the compute behind the evaluator layer. That makes the scoring stack more defensible, reduces platform dependency, and gives investors a clearer asset base than a company built only on rented inference.

          

          

            
Outcome

            
Control as the destination

            
The strongest long-term architecture gives Norynthe direct control over the evaluator layer and a credible path toward broader model independence over time. Hardware is not just infrastructure here; it is part of the company's defensibility.
